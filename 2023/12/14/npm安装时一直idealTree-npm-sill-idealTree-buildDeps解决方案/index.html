npm安装时一直卡在idealTree:npm: sill idealTree buildDeps解决方案

Created2023-12-14|Updated2023-12-18|study
|Post Views:

背景

使用npm安装hexo-butterfly-extjs时一直卡着不动，命令行显示

报错原因

可能原因：

打开cmd窗口，输入命令：npm config get registry，查看当前镜像地址
默认地址（https://registry.npmjs.org/）在国外，所以安装缓慢或卡顿。

不过也查到存在直接换源无效的案例

解决方法

参考：https://blog.csdn.net/kelly0721/article/details/121908256

  1. 删除.npmsrc

    mpmsrc详解：https://blog.csdn.net/kelly0721/article/details/121908256

    .npmrc，可以理解成npm running cnfiguration, 即npm运行时配置文件。简单点说， .npmrc 可以设置 package.json 中依赖包的安装来源，既从哪里下载依赖包。

    通过如下指令获得用户配置文件目录， 删除.npmrc文件

  2. 清除缓存（会出现警示）

    npm cache clean –force

  3. 更换淘宝源

    默认链接：https://registry.npmjs.org/ （可同理用npm config set registry “url” 重置）
